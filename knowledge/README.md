# Knowledge Base (Curated, Stable)

This folder contains **curated knowledge** that should remain stable over time: domain rules, invariants, reference tables, and canonical examples.

It is not a scratchpad.

## What belongs here

- Canonical billing domain references (glossary excerpts, invariants summaries)
- Stable reference tables (currency/rounding rules we support, time boundary conventions)
- Canonical example scenarios (inputs → outputs) that we want to preserve
- “How it works” references that should not change frequently

## What does NOT belong here

- Meeting notes
- Temporary investigations
- Unreviewed AI output
- Speculation or TODO lists

## Source of truth rule

Code and `docs/` remain the source of truth. `knowledge/` exists to make it easy for agents to retrieve stable context quickly and consistently.
